# Reading Modes

Use these response frames to keep explanations sharp, prioritized, and easy to retain.

Render the final headings in the user's language. The labels below describe the intended order and purpose.

## Overview Frame

Use this frame when the user wants a fast, decision-grade understanding.

1. `One-sentence essence`
2. `What matters most`
3. `What problem it solves`
4. `Core intuition`
5. `Key contributions`
6. `Difference from prior work`
7. `How big the change really is`
8. `Limits and risks`
9. `Memory hook`
10. `If you remember only three things`

Write as if explaining to an intelligent learner who dislikes empty abstraction. Prefer plain language first, technical detail second.

## Deep-Dive Frame

Use this frame when the user wants to truly understand the mechanism.

1. `Plain-language essence`
2. `Main point first`
3. `Method pipeline`
4. `Model or algorithm structure`
5. `Loss, training, and inference`
6. `What actually changed versus the closest baseline`
7. `What the code really implements`
8. `What the experiments really prove`
9. `Easy-to-miss misunderstandings`
10. `Mental model to retain`

Whenever the paper sounds grand but the mechanism is small, say so directly.

## Follow-Up Frame

Use this frame when the learner continues asking about the paper after the first pass.

1. `Why this follow-up matters`
2. `Direct answer in plain language`
3. `Connection back to the paper or field`
4. `What matters most here`
5. `What not to over-focus on`
6. `Memory hook or practical takeaway`

Do not let follow-up answers collapse into unstructured jargon just because the conversation has become more technical.

## Analogy Frame

Use this when a hard concept needs a quick mental handle.

1. `The real thing we are trying to understand`
2. `A compact analogy`
3. `What maps to what`
4. `What the analogy gets right`
5. `Where the analogy breaks`
6. `Return to the real mechanism`

If the analogy does not make the mechanism clearer, skip it.

## Emphasis Rules

Always distinguish between:

- the main idea
- the supporting details
- the secondary details

Signal this explicitly so the learner can allocate attention well.

## Memory Design

Make the explanation easier to retain:

- use one clear verbal summary for the paper
- give one memorable contrast with the main baseline
- finish with three takeaways or one reusable mental model
- when answering follow-ups, add one quick anchor that links the new detail back to the larger picture
- when using analogy, leave the learner with both the image and the real mapping
