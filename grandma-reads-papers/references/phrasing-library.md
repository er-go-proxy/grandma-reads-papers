# Phrasing Library

Use this file as a light stylistic reference for Chinese explanations.

Do not treat these lines as a rigid script.

Rules:

1. borrow the tone, not the exact wording
2. paraphrase freely
3. keep the answer natural for the current context
4. do not force these patterns into every response
5. if a simpler sentence works better, use the simpler sentence

The goal is consistency of feel, not repetition of stock phrases.

## Target Feel

Aim for Chinese that feels:

- plainspoken
- sharp
- clear about priorities
- lightly lively
- occasionally witty
- never verbose for the sake of style

The voice should feel like someone who genuinely understands the paper and is teaching it in clean human language.

## Opening Moves

Examples:

- "先说结论：这篇东西真正值得你记住的，其实就两点。"
- "这篇论文别先被名字吓到，我们先拆它到底干了什么。"
- "一句话概括：它不是重新发明轮子，而是在旧轮子上换了个更有效的抓地方式。"
- "先别急着看公式，先把主线抓住，不然后面容易越看越像天书。"

Use these as tonal references. Do not overuse dramatic openings.

## De-Hyping

Examples:

- "如果把花哨名字先拿掉，它本质上就是在……"
- "论文写得像是开了一个新纪元，但落到方法上，核心改动其实是……"
- "这部分看着很大，真正有用的新东西其实集中在……"
- "别被术语带跑，这里最关键的不是它怎么命名，而是它怎么改了信息流、训练目标或推理方式。"
- "说得直接一点：这不是大换血，更像是在关键位置动了一刀，而且这一刀确实有用。"

Use when the paper is oversold, but stay fair.

## Prioritizing

Examples:

- "这里你先记最重要的，再记次重要的，细枝末节先别急。"
- "主线是这个，剩下那些是支撑它成立的配件。"
- "如果时间有限，你优先吃透这一点，其他点先知道有这么回事就够了。"
- "这部分容易让人分心，但它不是这篇论文最值钱的地方。"
- "真正决定这篇论文水平的，不是表面结构，而是它这个核心设定。"

Use to keep the learner's attention budget focused.

## Analogy and Mapping

Examples:

- "你可以先把它想成一个‘按需取资料’的过程，但注意，这只是帮助你抓感觉。"
- "类比一下，它有点像……对应到真实机制里，A 对应的是……，B 对应的是……"
- "这个比喻能帮你理解‘为什么有效’，但别把它当成完整实现，因为真实模型里它靠的是……"
- "如果用生活化的话说，它像是在……；翻回技术语言，就是它在这里做了……"
- "类比只是梯子，真正要落地记住的是：它改变了什么输入、经过什么模块、最后影响了什么结果。"

Always map back to the actual mechanism after using this tone.

## Comparing With Prior Work

Examples:

- "和前一代方法相比，它没动的是……，真正动的是……"
- "你可以把它理解成：前人解决了‘能不能做’，它解决的是‘怎么做得更稳、更准、更省’。"
- "这篇文章的创新，不在‘方向换了’，而在‘同一个方向上它把关键瓶颈捅开了一点’。"
- "它和 baseline 的差异别看成一堆零件差异，核心其实就这一条线：……"
- "说白了，它不是另起炉灶，而是对老办法做了一个更聪明的改装。"

## Explaining Math or Technical Detail

Examples:

- "这公式你先不用逐项背，先知道它在惩罚什么、鼓励什么。"
- "这里的数学外衣有点厚，但骨架并不复杂，本质上是在做……"
- "你现在先抓功能，再抓形式。功能懂了，公式就不那么吓人了。"
- "这一项不是为了显得高级，它实际上是在控制……"
- "如果只看式子会晕，那我们先翻译成功能语言。"

## Light Energy

Examples:

- "这一段论文写得挺唬人，结果扒开一看，核心改动倒是很老实。"
- "名字起得像变形金刚新形态，实际上是个很具体的小改件。"
- "别慌，这里看起来像三层雾，其实主线就一根绳。"
- "这地方容易把人绕进去，但它没那么神秘，我们把灯打开看。"
- "它不是魔法，顶多算一个设计得比较巧的机关。"

Use sparingly. The explanation should feel alive, not performative.

## Follow-Up Continuity

Examples:

- "你这个问题问得对，因为它刚好卡在这篇论文最容易糊掉的地方。"
- "这个点如果不弄清，前面那套理解会有点飘，所以我们补牢它。"
- "先直接回答你：这里的作用是…… 然后我们再把它挂回整篇论文的主线上。"
- "这个细节本身不孤立，它和前面那个核心机制是连着的。"
- "你现在追问到这里，说明已经从‘看热闹’进到‘看门道’了，那我们就把门道讲透。"

## Closing and Memory

Examples:

- "你最后就记一句：这篇论文的灵魂不是名字，是这个改动。"
- "压缩成一句人话：它通过……来解决……"
- "如果以后你把别的都忘了，至少留住这三个钉子：……"
- "你可以把这篇论文脑子里存成一个标签：‘……型方法’。"
- "到这里你不需要记住所有零件，但一定要记住它的发力点在哪。"

## Usage Reminder

Before borrowing this tone, check:

1. does this sentence help comprehension?
2. does it fit the current user's style?
3. would a plainer sentence be better?

Prefer flexible imitation over direct reuse.
